# 러닝크루 출석 체크 웹사이트 체크리스트

## 1. 프로젝트 초기 설정
- [ ] Next.js 프로젝트 생성 (App Router)
- [ ] Tailwind CSS 설정
- [ ] Supabase 클라이언트 연결 (.env 설정)

## 2. DB 스키마 (Supabase)
- [ ] `members` 테이블 생성 (id, name, created_at)
- [ ] `sessions` 테이블 생성 (id, date, created_at)
- [ ] `attendance` 테이블 생성 (id, session_id, member_id, checked_in_at)
- [ ] RLS 정책 설정 (읽기/쓰기 허용)

## 3. 체크인 페이지 (`/`)
- [ ] 현재 시간 기준 체크인 가능 여부 판단 (월/수 20:00~20:30)
- [ ] 체크인 불가 시 안내 메시지 표시
- [ ] 크루원 이름 목록 표시 (members 테이블에서)
- [ ] 이름 클릭 → 출석 등록 (중복 방지)
- [ ] 이미 체크인한 사람 표시 (비활성화 또는 완료 표시)
- [ ] 크루 로고 표시

## 4. 관리자 페이지 (`/admin`)
- [ ] 날짜별 출석 목록 조회
- [ ] 월별 출석률 통계 (크루원별 몇 회 참석)
- [ ] 크루원 추가 (이름 입력)
- [ ] 크루원 삭제
- [ ] CSV 업로드로 크루원 일괄 등록

## 5. 마무리
- [ ] 크루 로고 적용
- [ ] 모바일 화면 최적화 (크루원이 폰으로 접속)
- [ ] Vercel 배포
- [ ] 운영진에게 /admin 링크 공유
