# 컨텍스트 노트 — 러닝크루 출석 체크 웹사이트

## 프로젝트 개요
러닝크루(약 70명) 월/수 정기 달리기 출석 체크 웹사이트.

## 핵심 결정 사항

### 인증 없음
운영진 페이지(/admin)는 별도 로그인 없이 URL 공유 방식으로 운영.
이유: 70명 소규모 크루, 운영진만 링크 알면 충분. 복잡도 최소화.

### 체크인 시간 제한
월요일, 수요일 오후 8:00~8:30 사이에만 체크인 가능.
이유: 달리기 시작 시간 8시, 늦참 허용 30분.
구현: 서버 시간 기준 한국 표준시(KST, UTC+9) 사용.

### 크루원 식별
이름 목록에서 클릭 방식 (직접 입력 아님).
이유: 실명 사용, 오타 방지, 70명이면 목록이 관리 가능한 범위.

### 기존 데이터 없음
새로 시작. 초기 크루원 데이터는 CSV로 일괄 등록 예정.

### 기술 스택 선택 이유
- Next.js App Router: 서버/클라이언트 컴포넌트 분리로 시간 체크 로직 서버에서 처리 가능
- Supabase: 사용자가 이미 계정 보유
- Tailwind CSS: 빠른 모바일 최적화 UI

## DB 설계 메모
- sessions 테이블: 매 달리기마다 row 하나. date는 YYYY-MM-DD 형식.
- attendance 테이블: session_id + member_id unique 제약으로 중복 체크인 방지.
- RLS: 클라이언트에서 직접 Supabase 호출하므로 anon key로 읽기/쓰기 허용 필요.

## 배포
Vercel 예정. 환경 변수: NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY
